# Self-Hosted Media Server Architecture (Jellyfin)

## Objective
Build a private media server for streaming across devices (Apple TV ecosystem).

## Problems Encountered
- Uncertainty around need for NAS vs PC
- Service integration confusion (Jellyfin role)
- Network accessibility considerations

## Troubleshooting Process
- Broke system into hardware, services, and networking layers
- Evaluated cost vs scalability tradeoffs
- Focused on control and flexibility

## Solutions
- Used existing PC as server (no NAS required initially)
- Installed and configured Jellyfin
- Designed setup for future expansion
- Integrated with Apple TV

## Result
- Fully functional local media server
- Expandable infrastructure
- Cost-efficient setup

## Time to Resolution
- Planning + setup: 2–3 hours
